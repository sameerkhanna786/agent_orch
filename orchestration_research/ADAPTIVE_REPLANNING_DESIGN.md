**Adapt-before-cut integration:**
The hook is a small ADAPT-BEFORE-CUT amendment to the governor's sterile-streak arm only — it does NOT change the no-progress frontier dual-AND or any hard cut. Each rung that runs decrements sarp_rungs_remaining. A frontier RISE during a rung resets sarp_rungs_remaining (via _reset_patience + a fresh episode budget) so genuine progress is not penalized; a rise also clears the sterile streak naturally (_observe context.py:765-766). Patience is made frontier-height-aware: SARP only intervenes when best_gold_passed/gold_total >= APEX_OMEGA_SARP_FRONTIER_FLOOR (e.g. 0.5 or an absolute min) — a low/trivial frontier gets NO extra patience and is cut on schedule (a stuck-from-the-start run is not coddled), while a near-complete run (mimesis 6146/6159 = 0.998) gets the bounded adaptive window.

**Termination bound:**
Cannot loop forever because every rung is bounded and the episode budget is finite and monotone-decreasing-without-progress: (1) sarp_rungs_remaining is a small integer (default sum of rung bounds, e.g. <= 6) decremented by each rung and NEVER incremented except on a genuine frontier RISE (a strict increase in best_gold_passed, which is itself bounded by gold_total). So either the frontier rises (bounded by the finite gold suite size) or rungs_remaining monotonically hits 0. (2) Each rung is individually capped (targeted repair <= SARP_TARGETED_MAX rounds; re-decompose/integrate/escalate = ONE each per episode, host-side seen-set keyed by residual sha so an identical residual never re-triggers). (3) diagnose_residual is cached per residual-set sha and runs at most N scouts per distinct residual — a re-entered identical residual is a cache hit, no new work. (4) The governor's hard cuts (nonresult-streak 8, harness-stall 8, agent-ceiling, opt-in token floor) are UNTOUCHED and always override SARP, so the absolute agent ceiling still bounds the whole run. (5) When the ladder is exhausted or diagnosis.stuck, control returns to should_continue_waves() which then cuts. Net: total extra agents added by SARP per cell <= (episodes) * (N diag scouts + bounded rung agents), and episodes are themselves bounded because each non-progressing episode permanently consumes the residual-sha seen-set entries.

## 3. The bounded adaptation ladder (concrete steps + numeric bounds)

Episode lifecycle: an **episode** is opened the first time `_sarp_observe` sees a sterile round at a non-trivial frontier with no active `_sarp_state`. On open: `_sarp_state = {"rungs_remaining": RUNGS_PER_EPISODE, "diag": None, "rungs_used_seq": [], "open_frontier": best_gold_passed}`. An episode is CLOSED (`_sarp_state=None`) on a genuine frontier rise (then normal repair resumes; the NEXT plateau opens a fresh episode) OR on terminal stuck/exhaustion.

Two hard counters bound everything (the adversarial-review required changes):
- `sarp_rungs_remaining` — per-episode pool, decremented by EVERY agent-dispatching rung (a/b/c/d) regardless of residual-sha, replenished ONLY by a genuine frontier rise (`advanced==True` AND `best_gold_passed > _sarp_state["open_frontier"]`).
- `_sarp_total_used` — per-RUN, NON-resettable, incremented by EVERY agent-dispatching rung; the run refuses any rung once `_sarp_total_used >= APEX_OMEGA_SARP_TOTAL_RUNG_BUDGET`. This is the single bound that makes the whole ladder terminate strictly EARLIER than `max_total_agents` and kills the sawtooth (trickle-progress re-arm).

Distinct-sha guard: a residual sha may re-arm the per-sha bounds (diag once, targeted ≤ TARGETED_MAX) only if `len(_sarp_seen_residual_shas) < MAX_DISTINCT_RESIDUALS`; otherwise the rung is refused and the ladder descends. Prevents the A-fails-for-B-fails mint-new-sha thrash (review finding 2).

| Rung | Trigger | Strategy | Distance signal | Bound |
|---|---|---|---|---|

### 3.1 Termination guards (resolving every adversarial-review concern)

These are MANDATORY (review verdict was "concern"; each maps to a required_change):

- **G1 (total ceiling, required_change #1):** `_sarp_total_used` is per-RUN, monotone, NEVER replenished by a frontier rise or a sha change. Every a/b/c/d rung increments it; rung refused at the cap. Guarantees the ladder terminates strictly before `max_total_agents` and kills the rise→reset→N-sterile-rungs sawtooth (review finding 3).
- **G2 (distinct-sha cap, required_change #2):** `_sarp_seen_residual_shas` is capped at `MAX_DISTINCT_RESIDUALS` per episode; a code-editing-but-non-progress repair that mints a new sha cannot re-arm diag+targeted indefinitely (review finding 2). The per-episode `rungs_remaining` pool is the primary bound; per-sha bounds are secondary and themselves capped by G2.
- **G3 (no streak-defuse on sterile rungs):** rung (a) is plain `repair_residual` and MUST NOT call `_reset_patience` (so a sterile (a)-rung lets `_sterile_streak` keep climbing toward 8 — exactly the prompt's terminating scenario). Only rungs (c)/(d) (which legitimately change strategy/model) may `_reset_patience`, and only ONCE each per episode.
- **G4 (harness-stall is never defused, required addendum):** `_reset_patience` (context.py:1614-1627) currently zeroes `_indeterminate_streak` (line 1627). SARP-triggered `_reset_patience` calls MUST NOT reset `_indeterminate_streak` — split a new `_reset_patience_for_switch(reset_indeterminate=False)` variant for SARP, OR have `_sarp_observe` snapshot `_indeterminate_streak` before and restore it after. This keeps `cut:harness-stall` (governor 119-120) reachable on a genuinely-broken harness (review finding 3, third path). Also cap SARP-triggered `_reset_patience` calls per episode at ≤2 (one for (c), one for (d)).
- **G5 (diag is read-only):** `diagnose_residual` uses `ctx.signals` (no plateau accounting, no `_observe`), so rung 0 can NEVER directly defuse a cut (review "what is actually sound" #1 — preserve it).
- **G6 (agent ceiling backstop, already sound):** every rung ultimately dispatches through `_engine.agent` (runtime.py:150-154) which returns `infra_nonresult` past `max_total_agents` — the unconditional hard floor under everything. SARP only ever terminates EARLIER.

```python

  "type": "object",
  "required": ["root_cause_class", "direction"],
  "properties": {
    "root_cause_class": {"type": "string", "enum": [
        "missing_shared_symbol", "semantic_logic_bug", "fixture_env_import",
        "coupling_integration", "unsolvable"]},
    "direction": {"type": "string"},                                  # qualitative WHICH-WAY
    "target_ids": {"type": "array", "items": {"type": "string"}},     # subset of residual to aim at
    "target_symbol": {"type": "string"},
    "target_files": {"type": "array", "items": {"type": "string"}},
    "sub_clusters": {"type": "array", "items": {"type": "array", "items": {"type": "string"}}},
    "stuck": {"type": "boolean"},
    "evidence_ids": {"type": "array", "items": {"type": "string"}},
    "reason": {"type": "string"},
  },
}
```

`diagnose_residual` flow (mirrors `diagnose` 1919-1993 and `goal_align_gate` 2129-2154):
1. Gate-off → `{}`. Cache hit on `residual_set_sha(residual_ids)` → cached.
2. Fan out N read-only scouts via `ctx.signals` (agent-id band disjoint from diagnose's 700700 and review's 700900 — use e.g. `701700 + sha%80000`). Each scout prompt includes: the still-failing node-ids, the `failure_excerpts` (assertion tails — the WHY), the implicated source/files (from the diff carry + repo_map), and a request for `root_cause_class + direction + target cluster + stuck`.
3. **Fact-check (anti-hallucination, exactly goal-gate 2138-2140):** `target_ids`/`evidence_ids` not in the real residual set are dropped; `stuck` is honored only on a GROUNDED majority (scouts citing residual-grounded evidence). An ungrounded `coupling_integration`/`unsolvable` verdict is downgraded to `semantic_logic_bug` (the safe targeted-repair default).
4. Aggregate by majority `root_cause_class`; union grounded `target_ids`/`target_files`/`sub_clusters`; `stuck = grounded_majority`.
5. Cache + return.

How diagnosis SELECTS the rung: residual COUNT says HOW FAR (drives `rungs_remaining` consumption + the non-trivial floor); diagnosis DIRECTION says WHICH WAY (selects (a) vs (b) vs (c) vs (e) per §3 triggers). It is never a fixed order.

How cause/direction is threaded to the targeted-repair agent: rung (a) calls `repair_residual(diag.target_ids or residual, excerpts=red["failure_excerpts"], prompt=<built from default residual brief + "\n--- SARP RE-AIM ---\nroot cause: {root_cause_class}\ndirection: {direction}\ntarget symbol: {target_symbol}\n">)`. `repair_residual` already accepts `excerpts=` (2412-2413) and `prompt=` (2415) — no signature change needed; SARP simply finally USES them.
